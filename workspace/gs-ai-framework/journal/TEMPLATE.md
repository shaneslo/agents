## Rules for this journal

1. No client-identifiable content, ever.
2. No firm-confidential content (internal tool names, internal system details) committed to a personal repo.
3. Honest failure notes. A journal that only logs wins is lying.
4. Write daily for the first 30 days. Then at minimum weekly.
5. Revisit monthly retrospectives in each 30/60/90 review.
6. If you skip a day, don't backfill. Just start again tomorrow.
